# FR-001 — dojo-tech-mobile-app (Field tech PWA)

## Scope

### 1. Job completion flow → Create Invoice (FR-001 #4)
- After checklist + photos + time entry are saved on the Complete Job screen, prompt "Create Invoice".
- Pre-fill line items from job_line_items (which now have price_book_id FKs after FR-001 #2).
- Tech can adjust quantity / add an item from the price book before invoicing.
- Tax auto-computes from price_book.tax_rate per line, summed.
- Submit calls connector /api/invoices with the job_id and adjusted line items.

### 2. Collect Payment in the field (FR-001 #4)
- After invoice creation, three options on the Payment screen:
  - **Send link** — tech texts/emails the GHL hosted payment URL to the customer.
  - **Card on file** — if customer has a saved card, tap to charge.
  - **Cash / Check** — tap method, enter amount, optional notes, mark paid. No processor — direct to payments table with method='cash'/'check', status='paid', collected_by={tech_id}, collected_at=now().
- Confirmation screen shows receipt summary + "Email receipt to customer" action.

### 3. Picker reuse where applicable
- Mostly read-only on the tech side except completion + payment.
- If any "edit job" screen exposes pickers, reuse the dispatch-board components.

### 4. Offline considerations
- If offline when collecting cash/check, queue the payment locally (IndexedDB) and sync when network returns.
- Card-on-file and send-link both require online — show clear error if attempted offline.

## Files most likely to touch
- CompleteJobScreen, PaymentScreen (new or expanded)
- Service worker — payment queue handling
- API client — invoice + payment endpoints
